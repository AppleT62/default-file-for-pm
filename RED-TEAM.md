# 독립 Red-Team 프로토콜

이 세계관의 확실성을 실제로 올리는 두 번째 경로. 지금까지의 모든 감사는 **같은 저자가
자기를 채점한 것**이라 자기 참조 편향에서 자유롭지 못하다. 아래 프롬프트를 **다른 세션
(가능하면 다른 모델)** 에 붙여넣어, 편향에서 벗어난 진짜 외부 시험을 돌린다.

## 사용법

1. **접근 경로**: 새 세션이 GitHub(`AppleT62/default-file-for-pm`)를 읽을 수 있으면
   **최신 회차 커밋이 있는 현행 브랜치**를 대상으로 준다(회차 58 현재
   `claude/folder-contents-review-rf2iwq`; 병합 후라면 기본 브랜치 최신). 안 되면
   `SYNTHESIS.md`와 `data/worldview.json` 내용을 붙여넣는다.
   **회차 58 정정(외부 보고서 #2가 실제로 겪은 문제)**: 이 항이 구판 브랜치
   (`claude/time-dimension-destiny-kdl77t`)를 하드코딩하고 있어 외부 감사가 구판
   (회차 44 시점)을 대상으로 수행됐다. 브랜치명은 갱신될 수 있으므로 **"최신 회차
   커밋"이라는 취지가 우선**한다 — 감사 시작 전에 대상의 `meta.last_cycle`을 확인하라.
2. **모델**: 이 루프를 주로 돌린 모델과 **다른 모델**로 실행하면 독립성이 높아진다.
   **다른 기반 모델에서 실행할 때는 [`external-redteam/`](external-redteam/) 키트를
   쓰라** — 저장소 접근 없이 첨부 파일만으로 감사 가능하도록 스냅샷·전용 프롬프트
   (공유 맹점 사냥 지시 포함)가 준비돼 있고, 완주 시 열린 문제 #18이 닫힌다.
3. 아래 프롬프트를 복사해 붙여넣는다.
4. 결과가 나오면 원 세션으로 가져와 반증 카드(대시보드 22부)와 대조하고,
   새 급소/기존 방어를 판정해 신념도에 사유와 함께 반영한다.

## 판정 외주 프로토콜 (회차 51 신설 — 열린 문제 #11)

감사자=피감사자 문제의 마지막 잔여는 "수용/하향 폭의 최종 결정을 저자가 내린다"는 것이다.
차기 감사부터 다음 절차를 적용한다:

1. red-team 보고서가 도착하면, 저자 세션은 **판정 없이** 결함 목록과 관련 데이터만 정리한다.
2. 별도의 **판정 세션**(감사 세션과도, 저자 세션과도 다른 새 세션)에 결함 목록 + 현재
   장부를 주고, 각 결함의 수용/기각과 **하향 폭(5포인트 격자)**을 결정하게 한다.
3. 저자 세션은 판정 결과를 **집행만** 한다. 기각하고 싶은 판정이 있으면 반론을 문서로
   남기되 수치는 판정대로 반영한다(반론은 차기 판정 세션의 입력이 된다).
4. 판정 세션의 결정 기록은 `worldview.json`에 판정자 표시와 함께 보존한다.

상태: **실사용 5회 완료(회차 53–58)**. 정직 공시(회차 57, RT-2 적발): ① 판정 세션의 판정문 원본 4건은 미보존 — 존재하는 것은 집행자(저자 세션)가 작성한 요약뿐이므로, 절차의 증거 사슬이 자기 증언으로 되돌아온다. 소급 불가. 이후 판정은 원문 보존 의무 — **첫 이행: 회차 58, 외부 보고서 #2 원문을 `redteam-archive/RED-TEAM-REPORT-2.md`로 보존.** ② 전 감사·판정 세션은 동일 기반 모델 — "다른 모델" 권고는 여전히 미이행(열린 문제 #18, 회차 58에도 미해소 명기).

### 회차 43 판정 요지 (원 세션)
