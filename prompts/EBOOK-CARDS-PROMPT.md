# 지시서 — 「기울어진 주사위」 앱 전자책 카드 콘텐츠 집필

당신은 이 저장소(`AppleT62/default-file-for-pm`)의 세계관을 **앱 전자책의 카드 콘텐츠**로 집필하는 작업을 맡는다. 설계는 이미 `APPDESIGNPROMPT.md` §2에 확정돼 있고, 당신의 일은 그 목차·규칙에 맞춰 **각 장의 카드 실물 텍스트를 쓰는 것**이다.

## 0. 먼저 읽을 것 (순서대로)
1. `APPDESIGNPROMPT.md` §0–§2 — 앱 정의·세계관 탭 목차(12장)·카드 변환 규칙. **이것이 당신의 스펙이다.**
2. `SYNTHESIS.md` — 종합 논고(§0–§10·부록 4편). **카드의 1차 원천.**
3. `index.html` 대시보드 텍스트, `data/worldview.json`(신념도·이원 회계), `RED-TEAM.md`·`redteam-archive/`(9장 원천).
4. `book/WRITING-PROMPT.md` — 참고만. **그 '책' 원고를 재사용하지 말 것.** 앱 전자책은 톤·구조·갱신 주기가 다른 별개 산출물이다.

## 1. 산출물
- 파일: `app-content/cards.json`(권장) 또는 장별 `app-content/ch{00..11}.md`.
- 카드 스키마(각 카드): `{ id, chapter, order, title, body, source_ref, credence?, column, lock, visual? }`
  - `source_ref`: 원천 문서·섹션·문단(예: `SYNTHESIS.md#§4-3문단`). **모든 카드 필수.**
  - `credence`: 경험 주장이면 `worldview.json`에서 읽은 신념도(하드코딩 금지 — 값 자체를 파일에 박지 말고 "worldview.json의 어느 키"인지 참조로 표기). 기능 열 항목은 `column:"functional"` + 라벨 "틀 채택 평가"(% 금지).
  - `column`: `"probability"` | `"functional"`.
  - `lock`: 무료 구간(프롤로그+1장 ~10화면)은 `false`, 나머지 `true`(§2.4 비즈니스 모델).

## 2. 집필 규율 (APPDESIGNPROMPT.md §2.2 준수)
- **1카드 = 1아이디어**, 본문 80–300자. 제목 ≤20자.
- **쉬운 언어**: 중학생 가독성. 전문어 첫 등장 시 1줄 풀이(예: "신념도 — 이 주장을 얼마나 믿을 만한지의 %, 측정값이 아니라 정직한 자기 평가"). 은유 적극 사용 후 "정확히 말하면…" 한 줄로 원뜻 보존.
- **무손실**: `SYNTHESIS.md`·`PROTOCOL.md`의 모든 문단이 최소 1개 카드에 매핑돼야 한다(커버리지 100%). 발췌 챕터는 문서화된 예외로 표기. 매핑표(`app-content/coverage.md`)를 함께 제출.
- **정직성 장치(협상 불가)**:
  - 경험 주장 카드에 신념도, 기능 열은 "틀 채택 평가" 라벨. 두 열 혼용 표기 금지.
  - "가설(검증 대기)·한계·경계 조건"은 확신 문장으로 승격 금지.
  - **금지 문구**: "과학적으로 증명된", "인생이 바뀝니다", "반드시/누구나 ~된다", 질병·우울·불안 치료 효능 암시.
  - 5장(운명애)은 경계 조건 2장 필수. 9장(자기 재판)·10장(열린 문제)은 (c) 판정·공유 맹점 확증을 **미화 없이** 그대로.

## 3. 목차 (APPDESIGNPROMPT.md §2.1 그대로 — 독자 인식 순서)
0 프롤로그 / 1 나는 확률의 구름이다 / 2 네 개의 층 / 3 시간 / 4 자유 / 5 운명애 / 6 관계·사회 / 7 삶의 부록 / 8 감정과 의미 / **9 이 책이 스스로를 재판한 기록(★차별점)** / 10 우리가 아직 모르는 것 / 11 그래서, 당신의 66일.
→ **9·10장은 삭제·약화 절대 금지.** 이 앱의 유일한 차별점이다.

## 4. 하지 말 것
- 원문에 없는 주장·성과 약속 신설 금지. 신념도 수치 임의 변경 금지(worldview.json이 진실 원천).
- 원문 문단 누락 금지(커버리지가 심판). 미화·과장 금지.
- '책' 원고 복사 금지 — 앱 카드는 독자적으로 새로 쓴다.

## 5. 작업 순서 제안
1. `SYNTHESIS.md`·대시보드를 문단 단위로 인덱싱 → 커버리지 표 초안.
2. 목차 장별로 카드 초안 집필(무료 구간 0·1장 먼저 — 앱 M1 우선).
3. 신념도·열·source_ref·lock 메타 채움.
4. 커버리지 100% 검증 + 금지 문구 셀프 린트.
5. `app-content/`에 저장하고 커버리지표와 함께 제출.
