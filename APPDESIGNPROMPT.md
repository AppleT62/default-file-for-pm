## 0. 이 앱의 한 문장 정의

77회차의 자기 검증(red-team 3개 개발사·판정 외주 23회·F 지수 자기 채점)을 거친 "운명 = 조건부 확률 분포" 세계관을 **책처럼 읽는 [세계관] 탭** + 그 이론의 세 실험을 **개인이 실측·기록하고 주간 리포트를 받는 [실행] 탭**으로 이루어진 앱. 사용자의 기록은 **동의 하에** 소유자에게 집계되어 연구·개선에 쓰이며, 그 대가로 사용자는 세계관과 연결된 개인 리포트를 받는다.

**차별화 전략 = 정직성 (협상 불가·상품의 핵심)**: 이 앱은 "인생이 바뀝니다"라고 말하지 않는다. "이 주장의 신뢰도는 60%이고, 세 AI가 이 이론을 '반증 불가능'으로 판정했으며, 여기가 반증 조건이다"라고 말하는 **유일한** 자기계발 콘텐츠다. 이 톤을 훼손하는 어떤 설계도 이 앱을 파괴한다.

## 2. [세계관] 탭 — 앱으로 만든 한 권의 책

### 2.1 목차 — 독자 인식 순서 (문서 논리 순서가 아님)

원천을 **독자가 몰입하기 좋은 순서**로 재편성한다: 훅 → 핵심 개념 → 확장 → 실천 연결 → **정직의 클라이맥스(자기 재판)** → 열린 결말. 목차는 무료 사용자에게도 전체 공개하되 잠긴 장에는 자물쇠+카드 수 표시.

| # | 장 | 원천 | 핵심 |
|---|---|---|---|
| 0 | **프롤로그 — 미래는 기울어져 있다** | 대시보드 0부·§0 | 5분 훅. "정해졌다 vs 바꿀 수 있다"의 제3의 답 |
| 1 | **나는 확률의 구름이다** | §1 (주파수=조건부 분포) | 핵심 은유: 고정된 성격이 아니라 상황마다 다른 반응의 지도 |

| 3 | **시간 — 정해진 것과 바꿀 수 있는 것** | §3 (블록 우주·불변량·성장 블록 철회) | "미래는 정해졌나"의 물리 |

### 2.2 카드 변환 규칙 (콘텐츠 파이프라인)

- **1카드 = 1아이디어**, 본문 80–300자. 제목(≤20자) + 본문 + 필요시 시각 요소 1개(수식·표는 스타일된 컴포넌트).
- **쉬운 언어 (book/WRITING-PROMPT.md의 규율 준수)**: 중학생 가독성. 전문어 첫 등장 시 1줄 풀이("신념도 — 이 주장을 얼마나 믿을 만한지의 %, 측정값이 아니라 정직한 자기 평가"). 은유 적극 재사용, 은유 뒤 "정확히 말하면…" 한 줄로 원뜻 보존.
- **무손실 요구 ("놓치는 문장이 없도록")**:
  1. 모든 카드에 `source_ref`(원천 문서·섹션·문단 ID).
  2. 각 카드에 **"원문 보기" 토글** — 해당 원문 문단 전체를 하단 시트로.
  3. 빌드 시 **커버리지 검증 스크립트**: SYNTHESIS.md·PROTOCOL.md의 모든 문단이 최소 1개 카드에 매핑되는지 검사, 미매핑을 CI 실패 처리(발췌 챕터는 문서화된 기준으로 예외 등록).
- **정직성 장치 (협상 불가)**:
  - 경험 주장 카드에 **신념도 뱃지**(예: `신뢰도 65%` 칩). 수치는 `worldview.json`에서 빌드 타임에 읽음 — **하드코딩 금지**(장부 갱신 시 앱도 갱신).
  - **이원 회계 표시**: 이 장부는 확률 열(관측이 움직임)과 기능 열(틀 채택 평가·손익 회계가 움직임)로 나뉜다. 기능 열 항목(공리 I·II·III·V·VIII·주파수·기술의 공리 등)은 % 대신 **"틀 채택 평가"** 라벨로 표기(원문 규약). 두 열을 섞어 표기 금지.
  - "가설(검증 대기)·한계·경계 조건"은 카드에서도 같은 지위로 — 확신 문장 승격 금지.
  - **금지 문구**: "과학적으로 증명된", "인생이 바뀝니다", "반드시/누구나 ~된다", 질병·우울·불안 치료 효능 암시. (금지 문구 린트 CI.)
  - 5장(운명애) 경계 조건 2장 필수, 9장은 (c) 판정·공유 맹점 확증을 **미화 없이** 그대로.

### 2.3 카드 뷰어 UX

- **세로 스와이프** = 카드 이동(쇼츠 문법), **가로 스와이프** = 장 이동. 상단 진행바(`3/24`), 하단 장 제목 pill.
- 북마크(길게 누르기), 공유(카드를 이미지로 렌더 — 워터마크에 앱명+신념도 뱃지), "원문 보기" 토글.
- 읽던 위치 자동 저장, 목차에 장별 진행률 링. 다크/라이트, 글자 3단계, 오프라인 캐시.

### 2.4 페이월 (선택 — 비즈니스 모델에 따라)

무료: 프롤로그+1장 도입(카드 1–10) + 전체 목차 + [실행] 탭의 안전 게이트·0단계. 페이월 카피는 정직하게: "우리는 결과를 약속하지 않습니다 — 신뢰도 수치와 반증 조건, 그리고 이 이론이 세 번 얻어맞은 기록까지 함께 팝니다." **단, 주간 리포트·데이터 교환은 페이월과 독립적으로 설계(§4·§5)** — 결제가 동의를 강제하는 구조 금지.

## 3. [실행] 탭 — 실험 A·B·C의 개인 실측 기록

**정직성 최우선 라벨(모든 실행 화면 상단)**: "이것은 당신 한 사람의 실측(N=1)입니다 — 이론을 증명하지 않습니다. 확증 실험(수백 명 규모)은 연구자의 몫이고, 여기서 당신이 얻는 것은 '이 방법이 *당신에게* 듣는가'에 대한 정직한 답입니다." (원천: EXPERIMENTS.md·prereg — 실험 C만 개인용, A·B는 연구자 규모임을 원문이 명시.)

### 3.1 세 실험의 개인용 각색

| 실행 | 원천 | 개인 각색 (앱에서 기록) | 기록 데이터 |
|---|---|---|---|
| **실행 C — 66일 편집** (주력) | 실험 C·PROTOCOL·PREREG-EXP-C | 단계 도입 ABA(기저선 2주→B₁ 3주 실행의도→B₂ 3주 수면게이트·중요도낮추기→철회 2주). "설치 vs 노력 유지" 구분 | if-then 일일 기록·주간 ○/△/✗·0층(수면·운동·햇빛)·기분/수면/기능 안전 체크·재측정 전후 |
| **실행 A — 운명애 글쓰기** | 실험 A·PREREG-EXP-A | 2주 격일 "분포 긍정 글쓰기"(실패를 '내 분포의 정상 꼬리'로 재기술+회복 가능성) + 자기방해 자가 관찰 | 회기별 글·자기방해 자가 척도(사전/사후) |
| **실행 B — 온도 축** | 실험 B·PREREG-EXP-B | 아이디어 과제를 고온(판단 유예)/저온(선별) 조건으로 자가 수행 + 다양성 자가 관찰 | 과제 산출물·조건별 자가 평정 |

- 진행 중 사이클 1개만(원문 "변수 하나"), 완료 후 새 사이클·이력 보관.
- **판정의 외주(원문 핵심)**: 실행 C 시작 시 성공 기준을 수치로 입력→ 66일 후 판정을 앱(규칙 기반)+선택적 제3자에게 위임. "내가 좀 나아진 것 같아"가 아니라 사전 기준 통과 여부.

### 3.2 안전 게이트 (필수·스킵 불가 — 원문 임상 게이트 1:1)

- **시작 전 게이트**(무료·상시 접근): 원문 문구 그대로 "2주 이상 지속되는 불면·무기력·흥미 상실·식욕 변화는 이 워크시트의 대상이 아니라 전문 평가의 대상입니다." + 지역 상담 연결(한국: 정신건강 상담 1577-0199, 자살예방 109 등).
- **자동 중단 규칙**(회차 72·보고서 #5 G6, 본인 재량 불가): 주간 체크에서 기분·수면·기능 중 **2개 영역 이상 동시 악화 즉시 중단** + 전문 평가 안내 / 1개 영역 2주 연속 악화 중단. 앱이 자동 감지·차단(사용자가 끌 수 없음).
- **방어적 비관주의 경계**: 온보딩 자가 문답 → 해당자에게 B₂ '중요도 낮추기'를 기본 해제(역효과 경고).
- 중단은 실패가 아니라 안전 게이트 작동으로 기록. 위기 신호 사용자의 데이터는 **연구 집계에서 제외**(§5).

## 4. 주간 리포트 엔진

사용자가 기록하면 **매주 1회** 세계관과 연결된 개인 리포트를 생성한다.

### 4.1 리포트 구성 (모든 리포트 공통)

1. **이번 주 실측 요약**: 준수율·이탈 횟수·0층 상태·패턴 변화 (사용자 데이터로 계산).
2. **세계관 연결**: 그 데이터를 이론 개념에 정직하게 연결. 예: "이번 주 if-then 준수율 78% — 편집 기제가 당신 분포에서 작동하는 초기 신호일 수 있습니다. 단 철회기 전까지는 '설치'인지 '2주 버티기'인지 구분할 수 없습니다." **신념도·한계 병기 의무.**
3. **안전 점검**: 기분/수면/기능 추세, 악화 신호 시 중단·전문 평가 안내.
4. **다음 주 한 걸음**: 원문 프로토콜의 다음 단계 안내.
5. **정직 각주**: "이 리포트는 당신 한 사람의 관측입니다. 이론을 증명하지도, 당신을 진단하지도 않습니다."

### 4.2 생성 방식 — 2계층

- **핵심(규칙 기반·항상 동작·오프라인 가능)**: 데이터 → 통계·패턴 → 템플릿 리포트. LLM 없이도 완결. 안전 판정은 **반드시 규칙 기반**(LLM에 위임 금지).
- **선택 계층(자체 LLM API·§6)**: 서사적 개인화가 필요하면 소유자의 LLM API 호출. 입력 = 사용자 요약 데이터 + 세계관 발췌. **프라이버시 안전장치 필수**: (i) LLM 전송 전 비식별화, (ii) LLM 처리에 대한 별도 동의, (iii) 금지 문구 린트를 LLM 출력에도 적용, (iv) 안전 판정은 LLM 출력을 신뢰하지 않고 규칙 기반이 최종.
- **금지**: 리포트가 결과를 약속하거나("좋아지고 있어요!"), 진단하거나, 확신으로 승격하는 것. 리포트도 세계관 카드와 같은 정직성 규율.

## 5. 정직한 데이터 교환 (리포트 ↔ 데이터) — 동의·프라이버시

**핵심 원칙(이 앱의 정체성)**: 리포트를 받기 위해 사용자는 데이터를 기록해야 한다 — **이건 강제가 아니라 필연**(데이터 없이는 리포트를 만들 수 없으므로). 그러나 **소유자의 2차 활용(연구·데이터셋 축적)은 별도의, 강제되지 않는 명시적 동의**로만 이루어진다. 이 분리가 법적 유효성(개인정보보호법·GDPR '자유로운 동의')과 브랜드 정직성을 동시에 지킨다.

### 5.1 데이터 흐름 (3단 동의 — 계층 분리)

| 계층 | 무엇 | 동의 | 리포트 조건? |
|---|---|---|---|
| **A. 리포트 생성** | 사용자 기록을 소유자 백엔드가 처리해 *그 사용자의* 리포트 생성 | 서비스 이용에 필수(계약 이행) — 옵트인 | 예 (필연) |
| **B. 연구·개선 활용** | 비식별·집계된 기록을 소유자가 세계관 연구·앱 개선에 사용 | **별도 옵트인**(기본 꺼짐·거부해도 리포트 받음) | 아니오 |
| **C. 개별 원본 심화 활용** | 개별 원본 기록(비집계)을 소유자가 심화 분석 | **별도의 강화된 명시 동의**(민감정보 별도 동의, 언제든 철회·삭제) | 아니오 |

- **민감정보 처리(개인정보보호법 §23)**: 기분·수면·정신 기능·심리 기록은 민감정보 — 계층 B·C는 **별도의 명시적 동의** 필수, 처리 목적·항목·보유기간 개별 고지.
- **비식별화 기본**: 계층 B는 집계·비식별 기본. 원본 개별 데이터(C)는 최소 수집·최소 보유.
- **위기 사용자 제외**: 안전 게이트가 발동한(악화·중단) 사용자의 데이터는 **연구 집계에서 자동 제외** — 취약 상태 데이터의 수집 금지.
- **철회·삭제·내보내기**: 언제든 동의 철회, 전체 삭제, CSV 내보내기. 철회 시 이미 비식별 집계된 것을 제외하고 개별 데이터 파기.
- **투명성 화면**: [내 정보]에 "당신의 데이터가 어디로 가는가" — 무엇이·왜·누구에게·얼마나 보관되는지 평이하게. 다크패턴(사전 체크·묶음 동의·미로 UI) 금지.

### 5.2 소유자 측 집계 파이프라인

- 동의(계층 B/C) 사용자의 기록 → 백엔드 → 비식별 처리 → 소유자 대시보드/데이터셋.
- **에피스테믹 정직(원문 규율 계승)**: 앱에서 모인 자기보고 집계는 **관측 데이터이지 사전등록 RCT가 아니다.** 소유자가 이 데이터로 "이론이 확증됐다"고 주장 금지 — 실험 A·B의 확증적 지위는 연구자 규모 사전등록 실행에만 있음(EXPERIMENTS.md 정직 고지). 집계는 가설 생성·패턴 탐색까지.

## 6. 기술 명세

- **플랫폼**: Flutter(iOS+Android 동시) 또는 React Native. 웹은 마케팅 랜딩+무료 미리보기.
- **콘텐츠 파이프라인**: 원천 MD/JSON → 빌드 스크립트가 카드 JSON(텍스트·source_ref·신념도·열(확률/기능)·잠금)으로 변환 → 앱 렌더. 콘텐츠 갱신은 원격 번들+버전 체크(앱 재배포 없이).
- **백엔드**: 인증(소셜)·구독 영수증 검증·콘텐츠 CDN·기록 백업(암호화)·**동의 관리 서비스**·**리포트 생성 잡(주간 스케줄러)**·**집계 파이프라인**. Supabase/Firebase 급 + 서버리스 함수.
- **자체 LLM API 연동(선택)**: 소유자 API 키를 서버 측에 보관(클라이언트 노출 금지), 리포트 선택 계층에서 호출. 요청 전 비식별화 미들웨어. 사용량·비용 상한. 안전 판정은 API에 위임 금지.
- **데이터 보안**: 기록은 민감 데이터 — 저장·전송 암호화, 접근 최소권한, 감사 로그. 마케팅·광고 활용 금지.
- **분석**: 카드 이탈·전환·사이클 완주율(메타데이터만, 기록 '내용' 수집은 계층 B/C 동의 하에만).
- **접근성**: 스크린리더 낭독, 터치 타깃 44pt, 명도 WCAG AA.

## 7. 법·안전 요구 (필수)

- **의료 부인**: "본 앱은 의료·심리치료가 아니며 진단·치료를 대체하지 않습니다." 온보딩+상시 접근.
- **동의**: 개인정보처리방침(항목·목적·보유기간·제3자), 민감정보 별도 동의(계층 B/C), 자동갱신·해지(스토어 정책), 청약철회.
- **강제 동의 금지**: 리포트·핵심 기능 접근을 계층 B/C 동의에 묶지 않는다(§5). 거부해도 리포트를 받는다.
- **연구 인용**: 원문 한정 표기("단일 RCT"·"논쟁 중"·"(c) 반증 불가 판정") 유지.
- **미성년·취약**: 연령 고지, 위기 신호 시 데이터 수집 중단·전문 연결.

## 8. 수용 기준 (완료 정의)

1. 무료 사용자: 온보딩→안전 고지→프롤로그+1장(카드 10)→페이월→전체 목차 열람.
2. 유료 사용자: 전 장 열람 + 실행 사이클 1개 생성·기록·주간 리포트 수신·재측정 완주.
3. 커버리지 CI 통과(원문 문단 매핑 100%·발췌는 문서화 예외).
4. 경험 주장 카드에 신념도 뱃지 + 기능 열은 "틀 채택 평가" 라벨 + "원문 보기" 동작.
5. 안전 게이트·자동 중단·역효과 경계가 지시대로 동작(QA 시나리오), 위기 사용자 데이터 집계 제외.
6. 금지 문구 린트 CI(카드+리포트+LLM 출력).
7. 3단 동의(A/B/C) 분리·철회·삭제·내보내기 동작, 계층 B/C 거부해도 리포트 정상.
8. 주간 리포트: 규칙 기반 단독으로 완결 생성(LLM 없이), 안전 판정 규칙 기반 확인.
9. 스와이프 60fps, 오프라인 구매 콘텐츠 열람.
