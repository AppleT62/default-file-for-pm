# 앱 설계·구현 프롬프트 — 「기울어진 주사위」 (가칭)

> 이 문서는 개발자(또는 AI 코딩 에이전트)에게 그대로 전달하는 **설계 지시서**다.
> 원천 콘텐츠 저장소: `AppleT62/default-file-for-pm` (브랜치 `claude/folder-contents-review-rf2iwq`)
> — `SYNTHESIS.md`(종합 논고), `index.html`(대시보드 0–33부), `PROTOCOL.md`(66일 워크시트),
> `EXPERIMENTS.md`, `data/worldview.json`(신념도 장부).

---

## 1. 역할과 목표

너는 이 프로젝트의 **프로덕트 설계·구현 담당**이다. 아래 명세대로 자기계발 카테고리의
모바일 앱을 설계·구현하라.

**한 문장 정의:** 57회차의 검증(red-team 11팀·판정 외주 4회)을 거친 "운명 = 조건부 확률
분포" 세계관 보고서를 **한 권의 책처럼 스와이프하며 읽는 앱** + 그 이론의 실천 프로토콜을
**기록·관리하는 도구**. 무료 맛보기 10카드 → 유료 결제 전환.

**비즈니스 목표:** 유료 구독자 확보. 전환 장치는 페이월이지만, **차별화 전략은 정직성**이다 —
이 콘텐츠는 "인생이 바뀝니다"라고 말하는 대신 "이 주장의 신뢰도는 60%이고, 여기가
반증 조건이다"라고 말하는 유일한 자기계발 콘텐츠다. 이 톤을 상품성의 핵심으로 취급하라.

## 3. 읽기 탭 — "앱으로 만든 한 권의 책"

### 3.1 콘텐츠 챕터 구조 (목차)

바로 이 두 챕터다. 마케팅 카피에도 이 지점을 사용하라.

### 3.2 카드 변환 규칙 (콘텐츠 파이프라인)

- **1카드 = 1아이디어**, 본문 80–300자. 제목(≤20자) + 본문 + 필요시 시각 요소 1개
  (수식·표·간단한 도식은 이미지가 아니라 스타일된 텍스트/컴포넌트로).
- **쉬운 언어 규칙:** 중학생이 읽을 수 있는 문장. 전문어는 첫 등장 시 1줄 풀이를 붙인다
  (예: "신념도 — 이 주장을 얼마나 믿을 만한지의 %"). 원문의 은유(찌그러진 주사위, 서랍장,
  운영체제/UI)는 적극 재사용.
- **무손실 요구사항 ("놓치는 문장이 없도록"):**
  1. 모든 카드에는 `source_ref`(원천 문서·섹션·문단 ID)가 있다.
  2. 각 카드에 **"원문 보기" 토글** — 탭하면 해당 원문 문단 전체가 하단 시트로 노출된다.
  3. 빌드 시 **커버리지 검증 스크립트**를 돌린다: SYNTHESIS.md·PROTOCOL.md의 모든 문단이
     최소 1개 카드의 `source_ref`에 매핑되는지 검사하고, 미매핑 문단 목록을 CI에서 실패로
     처리한다. (대시보드 8~30부는 챕터 9에 발췌 편성하되, 발췌 기준을 문서화한다.)
- **정직성 장치 (필수 — 협상 불가):**
  - 경험적 주장 카드에는 **신념도 뱃지**를 단다 (예: `신뢰도 70%` 칩). 수치는
    `worldview.json`에서 빌드 타임에 읽어온다 — 하드코딩 금지 (장부가 갱신되면 앱도 갱신).
  - 원문에서 "가설(검증 대기)", "한계", "경계 조건"으로 표시된 내용은 카드에서도 같은
    지위로 표시한다 — 확신 문장으로 승격 금지.
  - **금지 문구:** "과학적으로 증명된", "인생이 바뀝니다", "반드시/누구나 ~된다",
    질병·우울·불안의 치료 효능 암시. (RT-1 감사가 적발한 "구판 확신 팔기"의 재발 방지.)
  - 6장(운명애)에는 원문의 경계 조건 카드 2장을 반드시 포함: ① 역효과 경계(최악 시나리오를
    미리 그려서 잘해 온 사람은 이 기법이 역효과일 수 있음) ② 흡수 상태(되돌릴 수 없는
    실패는 긍정이 아니라 회피 대상).

### 3.3 카드 뷰어 UX

- **세로 스와이프** = 다음/이전 카드 (틱톡/쇼츠 문법). **가로 스와이프** = 챕터 이동.
  화면 상단에 챕터 내 진행 바(`3/24`), 하단에 챕터 제목 pill.
- 카드 전환은 스냅(페이지 단위). 프리로드 ±2카드. 60fps 유지.
- 카드 내 인터랙션: 북마크(길게 누르기), 공유(카드를 이미지로 렌더해 공유 — 워터마크에
  앱명+신념도 뱃지 포함), "원문 보기" 토글.
- 읽던 위치 자동 저장. 목차 화면에서 챕터별 진행률 링 표시.
- 다크/라이트 테마. 글자 크기 3단계. 오프라인 캐시(구매한 챕터).

### 3.4 페이월

- **무료 범위:** 카드 1–10 (프롤로그 전체 + 1장 도입) + 전체 목차 + 실천 탭의 0단계(게이트·0층 점검).
- 11번째 카드 위치에 **페이월 카드**: 남은 콘텐츠 요약(챕터 수·카드 수·실천 기능),
  가격, CTA 버튼. 톤은 콘텐츠와 동일하게 정직하게 — 권장 카피:
  > "여기까지가 무료입니다. 뒤에는 이 이론이 스스로 얻어맞은 기록(10장)과, 그래도
  > 살아남은 도구들(실천 탭)이 있습니다. 우리는 결과를 약속하지 않습니다 — 신뢰도
  > 수치와 반증 조건까지 함께 팝니다."
- 결제 모델: 구독(월/연) 우선, 평생 소장권 병행 검토. 스토어 IAP 규정 준수
  (iOS StoreKit2 / Android Billing). 구매 복원 필수.
- 잠긴 챕터 진입 시에도 첫 카드 1장은 미리보기로 노출 후 페이월.

## 4. 실천 탭 — 66일 워크시트의 앱화

`PROTOCOL.md`를 그대로 디지털화한다. **원문 단계 구조·문구·경고를 임의 삭제 금지.**

### 4.1 플로우 (원문 단계와 1:1)

1. **시작 전 게이트 (필수 통과 화면, 무료):**
   - 원문 문구 그대로: "2주 이상 지속되는 불면·무기력·흥미 상실은 이 워크시트의 대상이
     아니라 전문 평가의 대상입니다." + 지역 정신건강 상담 연결 안내(한국: 1577-0199 등).
   - 역효과 경계 자가 문답 1개(방어적 비관주의 성향 안내) — 해당자에게는 3단계
     "중요도 낮추기"를 선택 해제 기본값으로 제공.
   - 사이클 목표 1줄 입력 ("변수는 한 번에 하나" 규칙 안내).
2. **0단계 · 0층 점검:** 수면 시간·운동 횟수·햇빛 — 주간 반복 체크. 최소선 미달이 2주
   지속되면 게이트 안내를 재노출.
3. **1단계 · 진단 (첫 2주):** if-then 기록 — "상황 → 내가 한 것" 하루 3건 입력 UI
   (빠른 입력: 상황/행동 두 필드 + 시각 자동). 2주 후 앱이 반복 패턴 후보를 묶어 보여주고
   사용자가 시그니처 1–3개를 확정.
4. **2단계 · 설계:** 실행 의도 작성기 — "if [구체적 단서] → then [구체적 행동]" 템플릿,
   막연한 단서("스트레스 받으면") 입력 시 구체화 힌트. 최대 3개 제한(원문 규칙).
5. **3단계 · 가중:** 실패 예산 설정(66일 중 이탈 n회), 흡수 상태 지정(경고 문구 포함),
   격주 정밀도 감사(중요한 것 3가지 → 내 선택/주입 분류).
6. **4단계 · 운용 (9주):** 주간 체크(○/△/✗ + 한 줄 메모), 리마인더 알림(요일·시간 설정).
   66일은 "관행적 주기"라는 원문 고지 표시(18–254일 개인차).
7. **재측정 (66일 후):** 1단계와 동일 항목 재입력 → 전후 비교 화면.
   - "악화" 선택 시: 원문 지침 그대로 — 개입 중단 안내 + 기분·수면·기능 악화 동반 시
     전문 평가 안내 재노출.
   - "준수했는데 변화 없음" 선택 시: 원문 지침(단서 교체 1회 재설계 / "안 맞는다"도 유효한
     데이터) 분기 제공.
   - 선택 기능: **철회 2주 모드** (기록만 하고 규칙 미적용 후 재측정 — "설치 vs 노력 유지"
     구분, 원문 실험 C 설계).
8. **나의 판정:** "실행 의도로 내 행동 확률을 바꿀 수 있다"에 대한 사용자 자신의 신념도
   입력(시작 전/후 비교). 앱 문구: "이 숫자는 당신의 사적 기록입니다."

### 4.2 실천 탭 규칙

- 진행 중 사이클은 1개만(원문: 변수 하나). 완료 후 새 사이클 시작 가능, 이력 보관.
- 홈 위젯/알림: 오늘의 기록 리마인더, 주간 체크 리마인더.
- 데이터는 **로컬 우선 저장 + 선택적 클라우드 백업**(로그인 시). 기록은 민감 데이터로
  취급 — 암호화 저장, 마케팅 활용 금지, 내보내기(CSV)·전체 삭제 기능 필수.
- 유료 범위: 게이트+0단계는 무료, 1단계부터 유료 (읽기 탭과 동일 구독).

## 6. 기술 명세

- **플랫폼:** Flutter (iOS+Android 동시 출시). 대안 허용: React Native. 웹은 마케팅
  랜딩 + 무료 10카드 미리보기만.
- **콘텐츠 파이프라인:** 원천 MD/JSON → 빌드 스크립트가 카드 JSON(카드 텍스트,
  source_ref, 신념도, 잠금 여부)으로 변환 → 앱은 카드 JSON을 렌더. 콘텐츠 업데이트는
  앱 배포 없이 원격 구성(다운로드형 콘텐츠 번들 + 버전 체크).
- **백엔드:** 최소화 — 인증(소셜 로그인), 구독 영수증 검증, 콘텐츠 번들 CDN, 선택적
  기록 백업. Supabase/Firebase 급이면 충분.
- **분석:** 카드별 이탈 지점, 페이월 전환율, 실천 사이클 시작/완주율. 기록 '내용'은
  수집 금지(메타데이터만).
- **접근성:** 스크린리더 카드 낭독, 최소 터치 타깃 44pt, 명도 대비 WCAG AA.

- 구독 자동갱신 고지·해지 경로(스토어 정책), 개인정보처리방침(기록 데이터 항목 명시),
  청약철회 규정 준수.
- 콘텐츠 내 연구 인용은 원문의 한정 표기(예: "단일 RCT", "논쟁 중")를 유지한 채 표시.

## 8. 수용 기준 (완료 정의)

1. 무료 사용자: 온보딩 → 카드 10장 스와이프 → 페이월 도달 → 목차 전체 열람 가능.
2. 유료 사용자: 전 챕터 열람 + 실천 사이클 1개 생성·기록·알림·재측정까지 완주 가능.
3. 커버리지 스크립트 통과: 원천 문서 문단 매핑 100% (발췌 챕터는 문서화된 기준으로 예외 등록).
4. 모든 경험 주장 카드에 신념도 뱃지 렌더 + "원문 보기" 동작.
5. 임상 게이트·악화 분기·역효과 경계가 지시대로 동작 (QA 시나리오 포함).
6. 금지 문구 린트: 카드 JSON에 금지 문구 목록 검사 CI.
7. 스와이프 60fps, 오프라인에서 구매 콘텐츠 열람 가능.

## 9. 마일스톤 제안

1. **M1 (2주):** 콘텐츠 파이프라인 + 카드 뷰어 프로토타입 (챕터 0–1, 하드코딩 페이월)
2. **M2 (4주):** 전 챕터 변환 + 커버리지 CI + 페이월/IAP + 내 서재
3. **M3 (4주):** 실천 탭 전체 플로우 + 알림 + 백업
4. **M4 (2주):** 접근성·법 고지·분석·스토어 심사 대응 → 출시

## 10. 하지 말 것 (요약)

- 신념도·한계·경계 조건을 "마케팅에 방해된다"는 이유로 삭제하지 말 것 — 그것이 상품이다.
- 원문에 없는 성과 약속을 만들어 넣지 말 것.
- 실천 기록 데이터를 광고·추천에 쓰지 말 것.
- 임상 게이트를 스킵 가능한 광고 화면처럼 취급하지 말 것 (명시적 확인 필요).
- 카드 요약 과정에서 원문 문단을 누락하지 말 것 (커버리지 CI가 최종 심판).
