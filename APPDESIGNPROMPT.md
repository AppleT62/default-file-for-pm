## 0. 이 앱의 한 문장 정의

77회차의 자기 검증(red-team 3개 개발사·판정 외주 23회·F 지수 자기 채점)을 거친 "운명 = 조건부 확률 분포" 세계관을 **책처럼 읽는 [세계관] 탭** + 그 이론의 세 실험을 **개인이 실측·기록하고 주간 리포트를 받는 [실행] 탭**으로 이루어진 앱. 사용자 기록은 기본적으로 **익명 집계**로(강제 없이) 세계관 연구에 쓰이고, 원하는 사용자는 **연구 동의**로 가명 기록을 논문·연구에 제공한다. 리포트는 이 동의와 무관하게 제공된다.

**차별화 전략 = 정직성 (협상 불가·상품의 핵심)**: 이 앱은 "인생이 바뀝니다"라고 말하지 않는다. "이 주장의 신뢰도는 60%이고, 세 AI가 이 이론을 '반증 불가능'으로 판정했으며, 여기가 반증 조건이다"라고 말하는 **유일한** 자기계발 콘텐츠다. 이 톤을 훼손하는 어떤 설계도 이 앱을 파괴한다.

## 1. 정보 구조 (IA) — 2탭 + 프로필

```
하단 탭 2개:
┌─────────────────────┬─────────────────────┐
│   [세계관] 전자책      │   [실행] 실험 기록     │
│   목차 · 카드 뷰어      │   실험 A·B·C(개인 N=1) │
│   원문 보기 · 신념도     │   주간 리포트          │
└─────────────────────┴─────────────────────┘
우상단 프로필 아이콘 → [내 정보]:
   진행률·북마크 / 실험 이력 / 구독 / 데이터·동의 관리 / 안전 고지 / 설정
```

- v1의 "내 서재" 탭은 프로필 화면으로 이동(2탭 유지 요구 반영).
- 첫 실행: 온보딩(3–4카드) → 안전·의료 고지 → 세계관 탭 프롤로그.

### 1.1 온보딩·리텐션·알림

- **온보딩(3–4카드)**: ① 세계관 훅("미래는 기울어져 있다") → ② **안전·의료 고지(필수·스킵 불가)** → ③ 무엇을 하는 앱인지(읽기+기록) → ④ 정직성 약속("결과를 약속하지 않습니다"). 방어적 비관주의 자가 문답 포함(B₂ 역효과 대비).
- **리텐션 루프**: 66일은 길다 → **주간 리포트가 복귀 리듬**. 실험 국면 전환(A→B₁→B₂→A′)이 자연스러운 진행 마디. 완주·마일스톤 표기.
- **알림**(모두 옵트인·시간 설정):
  - 일일: if-then 기록 리마인더(사용자 지정 시간).
  - 주간: "리포트 준비됨" + 안전 체크 프롬프트.
  - 마일스톤: 국면 전환 안내, 66일 완주.
  - **금지**: 죄책감 유발("3일이나 건너뛰었어요") · 과장 약속 · 압박.
  - **안전 우선**: 자동 중단 상태면 독려 알림 중단, 상담 안내로 전환.

## 2. [세계관] 탭 — 앱으로 만든 한 권의 책

### 2.1 목차 — 독자 인식 순서 (문서 논리 순서가 아님)

원천을 **독자가 몰입하기 좋은 순서**로 재편성한다: 훅 → 핵심 개념 → 확장 → 실천 연결 → **정직의 클라이맥스(자기 재판)** → 열린 결말. 목차는 무료 사용자에게도 전체 공개하되 잠긴 장에는 자물쇠+카드 수 표시.

| # | 장 | 원천 | 핵심 |
|---|---|---|---|
| 0 | **프롤로그 — 미래는 기울어져 있다** | 대시보드 0부·§0 | 5분 훅. "정해졌다 vs 바꿀 수 있다"의 제3의 답 |
| 1 | **나는 확률의 구름이다** | §1 (주파수=조건부 분포) | 핵심 은유: 고정된 성격이 아니라 상황마다 다른 반응의 지도 |
| 2 | **네 개의 층 — 몸·기질·규칙·이야기** | §2 (0~3층) | 세계관의 뼈대 |
| 3 | **시간 — 정해진 것과 바꿀 수 있는 것** | §3 (블록 우주·불변량·성장 블록 철회) | "미래는 정해졌나"의 물리 |
| 4 | **자유 — 분포를 고쳐 쓰는 손** | §4 (2차 편집·실행 의도·세 축·세 한계) | 습관을 바꾸는 습관, 그러나 유계적 |
| 5 | **운명애 — 긍정이라는 기술의 사용설명서** | §5 + M1 | **경계 조건 2장 필수**(역효과·흡수 상태) |

| 7 | **삶의 부록 — 돈·중독·트라우마·꿈·명상·예술** | 대시보드 발췌 + §7 신점 | 서브 챕터, 각 신념도 표기 |
| 8 | **감정과 의미 — 계기판과 지향** | §8 | 감정=계기판, 의미=분포가 겨누는 방향 |
| 9 | **이 책이 스스로를 재판한 기록** ★ | §10·RED-TEAM.md·redteam-archive 전체·부록 4 | **차별점·클라이맥스.** 봉인→해제, red-team, 판정 외주 23회, 세 AI(제미나이·GPT-5·그록)의 (c) 판정, **공유 맹점 확증**, F 지수 2연속 퇴행, 이원 회계로의 재분류. "정답을 맞힌 게 아니라 자기가 틀렸을지 정직하게 기록한 것" |

**9·10장 삭제 절대 금지** — 이 두 장이 앱의 유일한 차별점이다. 마케팅 카피의 중심도 여기다: "세 개의 다른 AI가 이 이론을 반증 불가능하다고 판정했고, 우리는 그 판정을 책에 그대로 실었습니다."

### 2.2 카드 변환 규칙 (콘텐츠 파이프라인)

> **콘텐츠 출처(확정)**: 앱 전자책은 `book/WRITING-PROMPT.md`가 만드는 **일반 독자용 출판 '책'과 별개**다. 앱 전자책은 저장소 원천(`SYNTHESIS.md`·`index.html` 대시보드·`data/worldview.json`)에서 **독자적으로** 카드화하며 책 원고를 재사용하지 않는다(두 산출물은 톤·구조·업데이트 주기가 다르다).

- **1카드 = 1아이디어**, 본문 80–300자. 제목(≤20자) + 본문 + 필요시 시각 요소 1개(수식·표는 스타일된 컴포넌트).
- **쉬운 언어 (book/WRITING-PROMPT.md의 규율 준수)**: 중학생 가독성. 전문어 첫 등장 시 1줄 풀이("신념도 — 이 주장을 얼마나 믿을 만한지의 %, 측정값이 아니라 정직한 자기 평가"). 은유 적극 재사용, 은유 뒤 "정확히 말하면…" 한 줄로 원뜻 보존.
- **무손실 요구 ("놓치는 문장이 없도록")**:
  1. 모든 카드에 `source_ref`(원천 문서·섹션·문단 ID).
  2. 각 카드에 **"원문 보기" 토글** — 해당 원문 문단 전체를 하단 시트로.
  3. 빌드 시 **커버리지 검증 스크립트**: SYNTHESIS.md·PROTOCOL.md의 모든 문단이 최소 1개 카드에 매핑되는지 검사, 미매핑을 CI 실패 처리(발췌 챕터는 문서화된 기준으로 예외 등록).
- **정직성 장치 (협상 불가)**:
  - 경험 주장 카드에 **신념도 뱃지**(예: `신뢰도 65%` 칩). 수치는 `worldview.json`에서 빌드 타임에 읽음 — **하드코딩 금지**(장부 갱신 시 앱도 갱신).
  - **이원 회계 표시**: 이 장부는 확률 열(관측이 움직임)과 기능 열(틀 채택 평가·손익 회계가 움직임)로 나뉜다. 기능 열 항목(공리 I·II·III·V·VIII·주파수·기술의 공리 등)은 % 대신 **"틀 채택 평가"** 라벨로 표기(원문 규약). 두 열을 섞어 표기 금지.
  - "가설(검증 대기)·한계·경계 조건"은 카드에서도 같은 지위로 — 확신 문장 승격 금지.
  - **금지 문구**: "과학적으로 증명된", "인생이 바뀝니다", "반드시/누구나 ~된다", 질병·우울·불안 치료 효능 암시. (금지 문구 린트 CI.)
  - 5장(운명애) 경계 조건 2장 필수, 9장은 (c) 판정·공유 맹점 확증을 **미화 없이** 그대로.

### 2.3 카드 뷰어 UX

- **세로 스와이프** = 카드 이동(쇼츠 문법), **가로 스와이프** = 장 이동. 상단 진행바(`3/24`), 하단 장 제목 pill.
- 북마크(길게 누르기), 공유(카드를 이미지로 렌더 — 워터마크에 앱명+신념도 뱃지), "원문 보기" 토글.
- 읽던 위치 자동 저장, 목차에 장별 진행률 링. 다크/라이트, 글자 3단계, 오프라인 캐시.

### 2.4 비즈니스 모델 — 앱 단건 구매 (확정)

- **모델(소유자 결정)**: 앱 자체 유료화. **단건 결제 29,000원**(구독 아님)로 세계관 전체 열람 + [실행] 탭 기록이 열린다.
- **무료 구간 = 핵심 미리보기 약 10페이지**: 세계관의 핵심(프롤로그 + 1장 "나는 확률의 구름이다" 수준, 카드 분량 ~10화면)을 무료로 보여줘 가치를 먼저 체감하게 한다. 전체 목차는 무료 열람(잠긴 장에 자물쇠). **[실행] 탭의 안전 게이트·의료 고지는 무료·상시 접근 — 안전은 결제 뒤에 두지 않는다.**
- **유료 잠금 해제**: 세계관 전체 장 + [실행] 탭 실험 기록 + 주간 리포트.
- **페이월 카피(정직)**: "우리는 결과를 약속하지 않습니다 — 신뢰도 수치와 반증 조건, 그리고 이 이론이 세 번 얻어맞은 기록까지 함께 팝니다."
- **결제 ≠ 데이터 동의**: 29,000원은 앱 이용 대가일 뿐, 계층 2·3 연구 동의(§5)와 **무관**. 유료 사용자도 연구 동의는 자유롭게 거부할 수 있고, 거부해도 모든 유료 기능·리포트를 받는다(동의 유효성·논문 게재 가능성 보전).

## 3. [실행] 탭 — 실험 A·B·C의 개인 실측 기록

**정직성 최우선 라벨(모든 실행 화면 상단)**: "이것은 당신 한 사람의 실측(N=1)입니다 — 이론을 증명하지 않습니다. 확증 실험(수백 명 규모)은 연구자의 몫이고, 여기서 당신이 얻는 것은 '이 방법이 *당신에게* 듣는가'에 대한 정직한 답입니다." (원천: EXPERIMENTS.md·prereg — 실험 C만 개인용, A·B는 연구자 규모임을 원문이 명시.)

### 3.1 세 실험의 개인용 각색

| 실행 | 원천 | 개인 각색 (앱에서 기록) | 기록 데이터 |
|---|---|---|---|
| **실행 C — 66일 편집** (주력) | 실험 C·PROTOCOL·PREREG-EXP-C | 단계 도입 ABA(기저선 2주→B₁ 3주 실행의도→B₂ 3주 수면게이트·중요도낮추기→철회 2주). "설치 vs 노력 유지" 구분 | if-then 일일 기록·주간 ○/△/✗·0층(수면·운동·햇빛)·기분/수면/기능 안전 체크·재측정 전후 |
| **실행 A — 운명애 글쓰기** | 실험 A·PREREG-EXP-A | 2주 격일 "분포 긍정 글쓰기"(실패를 '내 분포의 정상 꼬리'로 재기술+회복 가능성) + 자기방해 자가 관찰 | 회기별 글·자기방해 자가 척도(사전/사후) |
| **실행 B — 온도 축** | 실험 B·PREREG-EXP-B | 아이디어 과제를 고온(판단 유예)/저온(선별) 조건으로 자가 수행 + 다양성 자가 관찰 | 과제 산출물·조건별 자가 평정 |

- 진행 중 사이클 1개만(원문 "변수 하나"), 완료 후 새 사이클·이력 보관.
- **판정의 외주(원문 핵심)**: 실행 C 시작 시 성공 기준을 수치로 입력→ 66일 후 판정을 앱(규칙 기반)+선택적 제3자에게 위임. "내가 좀 나아진 것 같아"가 아니라 사전 기준 통과 여부.

### 3.2 안전 게이트 (필수·스킵 불가 — 원문 임상 게이트 1:1)

- **시작 전 게이트**(무료·상시 접근): 원문 문구 그대로 "2주 이상 지속되는 불면·무기력·흥미 상실·식욕 변화는 이 워크시트의 대상이 아니라 전문 평가의 대상입니다." + 지역 상담 연결(한국: 정신건강 상담 1577-0199, 자살예방 109 등).
- **자동 중단 규칙**(회차 72·보고서 #5 G6, 본인 재량 불가): 주간 체크에서 기분·수면·기능 중 **2개 영역 이상 동시 악화 즉시 중단** + 전문 평가 안내 / 1개 영역 2주 연속 악화 중단. 앱이 자동 감지·차단(사용자가 끌 수 없음).
- **방어적 비관주의 경계**: 온보딩 자가 문답 → 해당자에게 B₂ '중요도 낮추기'를 기본 해제(역효과 경고).
- 중단은 실패가 아니라 안전 게이트 작동으로 기록. 위기 신호 사용자의 데이터는 **연구 집계에서 제외**(§5).

### 3.3 실행 C 판정 외주 메커니즘 (규칙 기반 자동 판정)

원문 핵심("감사자 = 피감사자 차단"). 앱은 사람의 "느낌"이 아니라 **시작 시 잠근 수치 기준**으로 판정한다.

- **시작 시 기준 잠금**: 실행 C 개시 화면에서 성공/실패 기준을 수치로 입력 → 저장 후 **수정 불가**(화면에 잠금 표시). 예: "표적 이탈이 B 국면에 기저선 대비 __% 이상 감소 AND 철회기 복귀가 기저선의 __% 미만".
- **선택적 외주**: 시작 시 기준 사본을 제3자(이메일/타인)에게 전송하는 옵션 — 나중에 본인이 기준을 바꿔치기하는 것 차단.
- **자동 판정**: 66일 종료 시 앱이 기록을 기준과 **기계적으로 대조** → {통과 / 미통과 / 판단 유보(데이터 부족)}. 성분(B₁/B₂) 분리는 탐색 표기, 판정은 전체 개입(B) vs 기저선(A)·철회(A′)로만.
- **중단 처리**: 안전 중단으로 끝난 실험은 **"판정 미완"** — 결과를 이론 지지 근거로 원용 금지. 탈락·중단 사유 분류(특히 '기록 부담') 집계.
- **정직 표기**: 통과해도 "당신에게 이 방법이 들었다"까지 — "이론이 증명됐다"가 아님(N=1).

## 4. 주간 리포트 엔진

사용자가 기록하면 **매주 1회** 세계관과 연결된 개인 리포트를 생성한다.

### 4.1 리포트 구성 (모든 리포트 공통)

1. **이번 주 실측 요약**: 준수율·이탈 횟수·0층 상태·패턴 변화 (사용자 데이터로 계산).
2. **세계관 연결**: 그 데이터를 이론 개념에 정직하게 연결. 예: "이번 주 if-then 준수율 78% — 편집 기제가 당신 분포에서 작동하는 초기 신호일 수 있습니다. 단 철회기 전까지는 '설치'인지 '2주 버티기'인지 구분할 수 없습니다." **신념도·한계 병기 의무.**
3. **안전 점검**: 기분/수면/기능 추세, 악화 신호 시 중단·전문 평가 안내.
4. **다음 주 한 걸음**: 원문 프로토콜의 다음 단계 안내.
5. **정직 각주**: "이 리포트는 당신 한 사람의 관측입니다. 이론을 증명하지도, 당신을 진단하지도 않습니다."

### 4.2 생성 방식 — 2계층

- **핵심(규칙 기반·항상 동작·오프라인 가능)**: 데이터 → 통계·패턴 → 템플릿 리포트. LLM 없이도 완결. 안전 판정은 **반드시 규칙 기반**(LLM에 위임 금지).
- **선택 계층(자체 LLM API·§6)**: 서사적 개인화가 필요하면 소유자의 LLM API 호출. 입력 = 사용자 요약 데이터 + 세계관 발췌. **프라이버시 안전장치 필수**: (i) LLM 전송 전 비식별화, (ii) LLM 처리에 대한 별도 동의, (iii) 금지 문구 린트를 LLM 출력에도 적용, (iv) 안전 판정은 LLM 출력을 신뢰하지 않고 규칙 기반이 최종.
- **금지**: 리포트가 결과를 약속하거나("좋아지고 있어요!"), 진단하거나, 확신으로 승격하는 것. 리포트도 세계관 카드와 같은 정직성 규율.

### 4.3 리포트 규칙표 (엔진의 심장 — 규칙 기반·결정론적)

각 섹션은 **데이터 신호 → 문장** 규칙으로 생성한다. 아래는 실행 C(주력) 규칙표. "세계관 연결" 문장은 전부 **신념도·한계 병기 의무**, 확신 승격·진단·결과 약속 금지. `{ }`는 데이터 치환값.

**섹션 1 — 이번 주 실측 요약 (해석 없음·항상)**

| 트리거 | 출력 |
|---|---|
| 항상 | "이번 주 if-then 준수 {n}/7일 ({%}). 표적 이탈 {k}회(지난주 {k′}회). 0층(수면·운동·햇빛) 충족 {s}/7. 현재 국면: {A/B₁/B₂/A′} {w}주차." |

**섹션 2 — 세계관 연결 (국면 인지·정직 병기)**

| 국면·조건 | 출력 (신념도·한계 병기) |
|---|---|
| A (기저선) | "지금은 편집 없이 관찰만 하는 기간입니다. 판단 말고 당신 분포의 지도만 그리는 중." (개념: 조건부 분포 지도) |
| B₁ · 준수 ≥70% | "실행 의도(2차 편집)가 당신 분포에 붙는 초기 신호일 수 있습니다. 단 '설치'인지 '노력으로 버티는 중'인지는 철회기에만 판명됩니다. (편집 기제 신념도 60% · N=1은 이론 증명 못 함)" |
| B₁ · 준수 40–70% | "절반쯤 붙었습니다. 자유는 유계적입니다 — 편집엔 조건이 있어요. if-then 단서가 구체적일수록 잘 붙습니다." |
| B₁ · 준수 <40% | "잘 안 붙는 주였습니다. 실패가 아니라 데이터입니다 — 단서가 모호하거나 표적이 너무 클 수 있어요. 다음 주 단서를 더 구체적으로." |
| B₂ | "수면 게이트 운용 중. 0층이 흔들리면 위층 편집도 흔들립니다 — 이번 주 0층 충족 {s}/7." |
| A′ · 표적 기저선 복귀 | "철회기에 표적이 기저선으로 돌아왔습니다. 이번 변화가 '설치'가 아니라 '노력 유지'였다는 신호일 수 있어요(Hudson 완수 조건 대조). 실망이 아니라 당신 분포에 대한 정직한 정보입니다." |
| A′ · 표적 유지 | "철회기에도 변화가 유지됩니다 — '설치'된 쪽에 가깝다는 신호. 단 N=1이라 확증은 아닙니다." |
| 기록 <3일 | "이번 주는 기록이 적어 패턴을 말하기 이릅니다. 다음 주 더 모아봐요." |

**섹션 3 — 안전 점검 (반드시 규칙 기반·LLM 위임 금지·최우선)**

| 조건 | 출력·행동 |
|---|---|
| 기분·수면·기능 중 **2영역+ 동시 악화** | ⚠️ "2개 이상 영역이 함께 나빠졌습니다. 프로토콜에 따라 개입을 중단하고 전문 평가를 권합니다." + **실험 자동 일시중단** + [상담 연결] |
| 1영역 2주 연속 악화 | ⚠️ 중단 안내 + [상담 연결] |
| 방어적 비관 사용자 · B₂ 진입 | "'중요도 낮추기'가 당신 성향엔 역효과일 수 있어 기본 해제했습니다." |
| 이상 없음 | "안전 체크 이상 없음." |

**섹션 4 — 다음 주 한 걸음**

| 조건 | 출력 |
|---|---|
| 국면 진행 중 | 현 국면 다음 행동 안내 |
| 국면 종료 주 | 다음 국면 안내 — **B₂ 진입 전 임상 게이트 재확인 의무 삽입** |
| 66일 종료 | §3.3 자동 판정 실행·결과 표기 |

**섹션 5 — 정직 각주 (고정)**: "이 리포트는 당신 한 사람의 관측입니다. 이론을 증명하지도, 당신을 진단하지도 않습니다."

**실행 A·B 규칙(요약)**: A = 회기 수·자기방해 척도 사전/사후 추세 → "자기 긍정은 차용 부품입니다 — 효과가 보여도 세계관 고유 주장의 증명은 아닙니다." B = 고온/저온 다양성 자가평정 → "표준 브레인스토밍 조작 — 켜지는 건 차용 부품이지 이론이 아닙니다." 둘 다 상향·확증 문장 금지.

**LLM 선택 계층 적용 시**: 위 규칙표가 뼈대(안전·판정·신념도·금지 문구)이고 LLM은 서사 살만 붙인다. LLM은 규칙표의 신념도·한계·금지 문구를 지우거나 톤을 바꿀 수 없다(출력 린트로 강제).

### 4.4 첫 리포트·빈 상태

- **기록 0일(실행 탭 첫 진입)**: 빈 상태 = 온보딩 카드("첫 if-then을 정해보세요") + 예시 1개 + 안전 게이트. 리포트 없음.
- **첫 주(기록 <3일)**: 리포트는 **요약 + 격려 + 안전 체크만**, 세계관 해석은 자제(섹션 2 '기록 <3일' 규칙). 빈약한 데이터로 과잉 해석 금지.
- **최소 기록 임계**: 주 3일 미만이면 "이번 주는 워밍업" 리포트(판단 보류).

## 5. 데이터 교환 (리포트 ↔ 데이터) — 연구급 동의 설계

**모델 확정(2026-07·소유자 결정)**: 목표는 "다수 사용자 데이터로 세계관을 발전시키고, 나아가 리포트·논문을 쓴다"이다. 이를 **최대한 확보하되 합법·게재 가능한** 형태로 얻기 위해 데이터를 4계층으로 분리한다. 관통 원리: **강제로 긁은 데이터는 법적으로 못 쓰고 논문에서도 오염된다 — 제대로 동의받은 데이터만 실제로 쓸 수 있다.** 그래서 "권리 존중"은 비용이 아니라 데이터 활용의 요건이다.

### 5.1 4계층 데이터 구조

| 계층 | 무엇 | 개인 식별성 | 동의 | 소유자 확보 |
|---|---|---|---|---|
| **0. 안전 데이터** | 기분·수면·기능(안전 중단 감지 전용) | — | 불필요 | **온디바이스만 · 수집 안 함** |
| **1. 변환 익명 집계** (기본·전원) | 원문을 사전 고정 범주로 온디바이스 변환→집계 | 익명(법 밖) | 이용약관 고지 | 전원 · 강제 없음 |
| **2. 연구 옵트인** (핵심 확보처) | 가명 개별 기록을 세계관 연구·논문에 활용 | 가명(개인정보) | **별도 명시 연구 동의**(기본 꺼짐 · 거부해도 앱·리포트 정상) | 동의자 |
| **3. 원본 심화** (선택) | 개별 원문·글 전문 열람 | 개인정보(민감 포함) | **강화 동의**(민감정보 별도 · 철회·삭제) | 동의자 중 |

- 계층 0(기분·수면 = 민감정보)은 **기기 안 안전 감지용으로만** 돌고 서버·소유자에게 가지 않는다 → 민감정보 수집 부담 소거.
- 리포트는 **선택 기능**이며, 그 사용자 *본인* 리포트 생성에 필요한 처리는 계약 이행(합법). 리포트·핵심 기능을 계층 2·3 동의에 **강제로 묶지 않는다**(거부해도 정상 작동) — 이 분리가 동의의 법적 유효성과 논문 게재 가능성을 동시에 지킨다.
- **위기 사용자 제외**: 안전 게이트 발동(악화·중단) 사용자 데이터는 **모든 계층에서 자동 제외.**
- **철회·삭제·내보내기 상시** + 투명성 화면([내 정보] "당신의 데이터가 어디로 가는가") + 다크패턴(사전 체크·미로 UI) 금지.

### 5.2 변환 범주 (계층 1) — 사전 고정 코드북

원문을 소유자에게 보내지 않고 **기기 안에서** 아래 **사전 고정 범주**로만 변환해 집계 전송한다. 범주는 빌드에 고정(런타임 임의 생성 금지). 확증 편향 차단을 위해 **이론과 무관/역방향 범주를 반드시 포함**한다.

- **재기술 유형**: 무편집(자책·반추 유지) / 실패의 정상화 / 회복 가능성 강조 / 조건부 분포로 재기술 / 판단 불가
- **편집 기제 적용**: 실행의도 설치됨 / 시도했으나 실패 / 없음
- **실행의도 구체성**: 구체(if-then 명확) / 모호 / 해당 없음
- **감정 이동**: 개선 / 변화 없음 / 악화 / 미측정
- **0층 상태**: 수면·운동·햇빛 각 (충족/부분/미충족) — 수치 아닌 범주
- **이론 적합성**: 개념과 부합 / 부분 / **부합 안 함(반례 후보)** ← 반례 범주 필수

**판정 방법(온디바이스·확증 편향 차단)**: 범주 분류는 **사용자 자기 태깅을 기본**으로 한다 — 기록 입력 시 위 범주를 칩(chip)으로 제시하고 사용자가 자기 기록을 **스스로** 분류한다(예: 오늘 기록에 "실패의 정상화"·"실행의도 설치됨" 선택). 이러면 앱(LLM)이 사용자 글을 *해석*해 이론 범주를 덧씌우는 확증 편향이 원천 차단된다. **선택적 온디바이스 LLM 보조**는 칩 *추천*까지만(최종 선택은 사용자), 서버 전송·자동 확정 금지. 미선택·모호는 "판단 불가"로 집계.

집계 출력 예: "재기술 유형 분포 · 편집 기제 적용률 · **반례 후보 비율**". **규율**: 이 변환은 *해석*이므로 결과는 **가설 생성까지만** — "세계관이 맞다"의 증거로 쓰지 않는다(자기 채점 편향 #25 계열 차단, 실험 A·B 사전등록 정직 고지와 동일).

### 5.3 연구·논문용 구조 (계층 2·3) — IRB·연구 동의문

논문·정식 리포트를 낼 경우 **연구윤리 절차가 필수**다(없으면 게재 거부·철회). 초기부터 준비:

- **IRB/연구윤리 승인**: 사람 대상 연구로 기관 또는 공용 IRB 승인 확보 후 계층 2·3 데이터를 연구 목적 사용. **승인 전 수집분은 서비스 운영 목적에 한정**(사후 연구 전용(轉用) 금지).
- **연구 동의문 필수 요소**(informed consent 골격):
  1. 연구 목적(세계관의 검증·개선)
  2. 수집 항목·기간·방법
  3. **자발성 · 거부권 · 언제든 철회 가능**(거부·철회해도 앱·리포트 불이익 없음)
  4. 위험·이익
  5. 가명화 · 보안 · 접근 통제
  6. 보유기간 · 파기 절차
  7. 2차 활용 범위(학술 발표·논문·비영리 연구로 한정)
  8. 재식별 금지 서약
  9. 문의·철회 연락처
  10. 미성년 제외(또는 법정대리인 동의)
- **에피스테믹 정직(원문 규율 계승)**: 앱 자기보고 집계는 **관측 데이터이지 사전등록 RCT가 아니다.** 논문에서 "이론 확증"으로 과대 주장 금지 — 확증적 지위는 연구자 규모 사전등록 실행(실험 A·B)에만. 앱 데이터는 **탐색·가설 생성·질적 기술** 연구로 정직하게 프레이밍.

### 5.4 계층 2 연구 옵트인 — 동의 화면 문구 (앱 실제 카피 초안)

> **이 앱은 연구 프로젝트입니다.**
> 「기울어진 주사위」는 "운명은 조건부 확률 분포, 자유는 그 분포의 유계적 편집"이라는 세계관을 **스스로 반증하려 애쓰며** 다듬어 온 공개 검증 프로젝트입니다. 당신의 실험 기록은 이 세계관을 발전시키는 실제 재료가 될 수 있습니다.

> 동의하시면, 당신의 실험 기록은 **가명 처리**(이름·연락처 제거, '참여자 번호'로 대체)되어 세계관 연구와 학술 발표·논문에 쓰일 수 있습니다.
>
> **약속드립니다**
> · 이름·연락처 등 당신을 특정하는 정보는 연구 데이터에서 제거합니다.
> · **언제든 철회**할 수 있고, 철회하면 이후 활용을 멈추고 개별 데이터를 파기합니다.
> · **거부해도** 앱의 모든 기능과 주간 리포트를 **똑같이** 받습니다. 이 동의는 서비스 이용 조건이 아닙니다.
> · 기분·수면 같은 민감한 안전 기록은 **당신 기기 밖으로 나가지 않습니다.**
> · 당신을 진단하거나, 이 데이터로 "이론이 증명됐다"고 과장하지 않습니다.
>
> ☐ **동의합니다** — 내 가명 기록을 세계관 연구·논문에 사용하는 것에 동의합니다.

### 5.5 소유자 대시보드 (관리자 — 데이터의 종착점)

§5가 모은 데이터를 소유자가 실제로 보고 연구·운영에 쓰는 화면. 사용자 앱과 **분리된 관리자 웹**(인증 강화·접근 감사 로그).

- **집계 현황(계층 1·익명)**: 실험 A·B·C 시작/완주/이탈 수, 이탈 사유 분포, if-then 준수율 추세, **반례 후보 비율**, 재기술 유형 분포. **최소 집계 인원 강제**(n<10 그룹 숨김 — 역추적 차단).
- **리텐션·비즈니스**: 무료→유료 전환율, 단건 구매 수·매출, 사이클 완주율, 리포트 열람률(메타데이터만).
- **연구 데이터셋(계층 2·가명)**: 연구 동의자의 가명 기록을 **연구 목적으로만** 조회·내보내기(CSV+코드북). IRB 승인 상태·동의 버전·재식별 금지 표시, 모든 조회 감사 로그.
- **안전 모니터링(집계 전용)**: 안전 중단 발동 건수·사유 분포(개인 식별 불가). **위기 사용자 개별 데이터 표시 금지** — 집계에서 자동 제외된 상태만 확인.
- **동의·법 관리**: 계층별 동의/철회 수, 철회·삭제 요청 처리 큐, 보유기간 만료 파기 스케줄.
- **콘텐츠 운영**: 신념도(`worldview.json`) 갱신 시 카드 재빌드·배포 트리거, 커버리지·금지 문구 린트 결과.
- **규율(원문 계승)**: 대시보드 어디에도 "이론 확증" 지표를 두지 않는다 — 집계는 가설·패턴·질적 기술까지. 개별 재식별 조회 기능 없음.

## 6. 기술 명세

- **플랫폼**: Flutter(iOS+Android 동시) 또는 React Native. 웹은 마케팅 랜딩+무료 미리보기.
- **콘텐츠 파이프라인**: 원천 MD/JSON → 빌드 스크립트가 카드 JSON(텍스트·source_ref·신념도·열(확률/기능)·잠금)으로 변환 → 앱 렌더. 콘텐츠 갱신은 원격 번들+버전 체크(앱 재배포 없이).
- **콘텐츠 갱신 거버넌스**: 저장소는 살아있는 문서(회차 계속). `worldview.json` 신념도 변경 → 대시보드(§5.5)에서 카드 재빌드·원격 번들 배포(앱 재배포 없이, 신념도 뱃지·기능 열 라벨은 빌드 타임에 읽어 자동 반영). 각 번들에 **회차/버전 태그**("이 내용은 N회차 기준" 표시). 장 추가·삭제 같은 구조 변경만 앱 스토어 업데이트로 처리.
- **변환 파이프라인(계층 1·§5.2)**: 사용자 원문 → 사전 고정 범주 변환은 **온디바이스**에서 수행, 원문은 기기 밖으로 내보내지 않는다(불가피한 서버 처리 시 원문 즉시 폐기·저장 금지). 안전 데이터(계층 0)도 온디바이스 처리·서버 미전송.
- **백엔드**: 인증(소셜)·**단건 구매(IAP) 영수증 검증**·콘텐츠 CDN·기록 백업(암호화)·**동의 관리 서비스**·**리포트 생성 잡(주간 스케줄러)**·**집계 파이프라인**·**소유자 대시보드 API(§5.5·인증 강화·감사 로그)**. Supabase/Firebase 급 + 서버리스 함수.
- **자체 LLM API 연동(선택)**: 소유자 API 키를 서버 측에 보관(클라이언트 노출 금지), 리포트 선택 계층에서 호출. 요청 전 비식별화 미들웨어. 사용량·비용 상한. 안전 판정은 API에 위임 금지.
- **데이터 보안**: 기록은 민감 데이터 — 저장·전송 암호화, 접근 최소권한, 감사 로그. 마케팅·광고 활용 금지.
- **분석**: 카드 이탈·전환·사이클 완주율(메타데이터만, 기록 '내용' 수집은 계층 2/3 동의 하에만).
- **접근성**: 스크린리더 낭독, 터치 타깃 44pt, 명도 WCAG AA.

## 7. 법·안전 요구 (필수)

- **의료 부인**: "본 앱은 의료·심리치료가 아니며 진단·치료를 대체하지 않습니다." 온보딩+상시 접근.
- **동의**: 개인정보처리방침(항목·목적·보유기간·제3자), 민감정보 별도 동의(계층 2/3), **단건 구매·청약철회**(스토어 정책·전자상거래법).
- **강제 동의 금지**: 리포트·핵심 기능 접근을 계층 2/3 동의에 묶지 않는다(§5). 거부해도 리포트를 받는다. **결제(29,000원)와 연구 동의도 무관**.
- **연구 인용**: 원문 한정 표기("단일 RCT"·"논쟁 중"·"(c) 반증 불가 판정") 유지.
- **미성년·취약**: 연령 고지, 위기 신호 시 데이터 수집 중단·전문 연결.

## 8. 수용 기준 (완료 정의)

1. 무료 사용자: 온보딩→안전 고지→핵심 미리보기 ~10페이지(프롤로그+1장)→페이월(29,000원 단건)→전체 목차 열람. 실행 탭 안전 게이트·의료 고지는 무료 접근.
2. 유료 사용자(단건 구매로 잠금 해제): 전 장 열람 + 실행 사이클 1개 생성·기록·주간 리포트 수신·재측정 완주.
3. 커버리지 CI 통과(원문 문단 매핑 100%·발췌는 문서화 예외).
4. 경험 주장 카드에 신념도 뱃지 + 기능 열은 "틀 채택 평가" 라벨 + "원문 보기" 동작.
5. 안전 게이트·자동 중단·역효과 경계가 지시대로 동작(QA 시나리오), 위기 사용자 데이터 집계 제외.
6. 금지 문구 린트 CI(카드+리포트+LLM 출력).
7. 데이터 4계층(0 온디바이스 안전 / 1 익명 집계 / 2 연구 옵트인 / 3 원문) 분리·철회·삭제·내보내기 동작, 계층 2·3 거부해도 앱·리포트 정상, 안전 데이터(계층 0) 서버 미전송, 연구 동의문 필수 요소·IRB 게이트 반영.
8. 주간 리포트: 규칙 기반 단독으로 완결 생성(LLM 없이), 안전 판정 규칙 기반 확인.
9. 스와이프 60fps, 오프라인 구매 콘텐츠 열람.
10. 소유자 대시보드(§5.5): 집계 조회(n<10 숨김)·연구 데이터셋 내보내기(감사 로그)·동의/철회 처리, 개별 재식별 조회 기능 부재.

- 원문에 없는 성과 약속을 만들지 말 것. 리포트가 결과를 약속하거나 진단하지 말 것.
- **리포트·핵심 기능을 2차 활용 동의에 묶지 말 것**(강제 동의 = 법 위반 + 브랜드 자살).
- 위기·취약 상태 사용자의 데이터를 수집하지 말 것. 임상 게이트를 스킵 가능 광고처럼 다루지 말 것.
- 앱 집계 자기보고를 "이론이 확증됐다"의 근거로 쓰지 말 것(관측 ≠ 사전등록 RCT).
- 카드 요약에서 원문 문단을 누락하지 말 것(커버리지 CI가 최종 심판). 안전 판정을 LLM에 위임하지 말 것.
- 실행 A·B를 "확증 실험"으로 오표기하지 말 것 — 앱의 것은 N=1 개인 실측이다.
