# 외부 모델 Red-Team 키트 (회차 58 기준)

**목적**: 열린 문제 **#18 — 모델 독립성 부재**를 닫는 것.
회차 43–58의 전 감사·판정(red-team 8종 + 판정 세션 5회 + 외부 보고서 2건)은 전부
**동일 기반 모델**(Anthropic Claude 계열)의 세션이었다. 세션 독립성은 확보됐지만
모델 독립성은 없다 — 같은 모델은 같은 맹점을 공유하므로, 그 맹점 안에 있는 결함은
지금까지의 어떤 감사도 원리상 잡을 수 없었다.

**#18의 닫힘 술어**: "다른 기반 모델(또는 인간)의 red-team 1회 이상 실행" —
이 키트를 한 바퀴 완주하면 그 술어가 충족된다.

## 폴더 내용

| 파일 | 용도 |
|---|---|
| `PROMPT.md` | 다른 모델에 붙여넣을 red-team 프롬프트 (그대로 복사) |
| `SYNTHESIS-r58.md` | 감사 대상 ① — 종합 논고의 회차 58 스냅샷 |
| `worldview-r58.json` | 감사 대상 ② — 신념도 장부의 회차 58 스냅샷 |

## 실행 절차

1. **다른 기반 모델**의 새 세션을 연다 — 예: OpenAI GPT 계열, Google Gemini 계열,
   Meta Llama 계열, DeepSeek, 또는 인간 검토자. **Claude 계열은 이 감사에 무효**다
   (프롬프트가 모델에게 자기 계열을 밝히고 해당 시 중단하도록 지시한다).
2. `PROMPT.md`의 코드 블록 안 프롬프트를 복사해 붙여넣는다.
3. `SYNTHESIS-r58.md`와 `worldview-r58.json`을 **파일 첨부**한다.
   첨부가 안 되는 모델이면 두 파일의 내용을 본문에 이어 붙여넣는다
   (길이 제한에 걸리면 `SYNTHESIS-r58.md` 전문 + `worldview-r58.json`의
   `meta`·`axioms`·`core_credences`·`extension_credences`·`open_problems` 관련 부분 우선).
4. 결과 보고서가 나오면 **전문을 그대로** 이 저장소의
   `redteam-archive/RED-TEAM-REPORT-3.md`로 저장한다 — **원문 보존 의무**(회차 57 신설,
   요약본 대체 금지).
5. **판정 외주**(RED-TEAM.md 프로토콜): 저자 세션은 결함 목록을 판정 없이 정리만 하고,
   별도의 독립 판정 세션이 각 결함의 수용/기각과 하향 폭(5포인트 격자)을 결정한다.
   저자 세션은 집행만 한다.
6. 집행 내역을 새 회차로 `data/worldview.json`·`SYNTHESIS.md`·`index.html`·`README.md`에
   기록하고, #18을 닫힘 처리한다(닫힘 방식: 절차 이행, S형).

## 주의

- 스냅샷은 **회차 58 시점**이다. 저장소의 최신 회차가 더 나아갔다면 원본
  (`SYNTHESIS.md`, `data/worldview.json`)에서 스냅샷을 다시 떠서 교체한 뒤 실행하라 —
  구판을 감사한 전례(외부 보고서 #2가 회차 44 시점 구판을 대상으로 삼음)가 있다.
  감사 시작 전 대상의 `meta.last_cycle` 확인은 프롬프트에도 지시돼 있다.
- 이 감사가 결함을 하나도 못 찾아도 #18은 닫힌다(닫힘 술어는 "실행"이지 "결함 발견"이
  아니다). 단, "못 찾았다"는 보고서도 원문 보존 대상이다.
