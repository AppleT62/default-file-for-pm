# 외부 모델 Red-Team 키트 (회차 62 기준)

**목적(회차 61 갱신)**: 원래 목적이던 **#18(모델 독립성 부재)은 이 키트의 첫 완주 —
보고서 #4(Gemini 3.1 Pro, `redteam-archive/RED-TEAM-REPORT-4.md`, 판정 (c)) — 로 회차 61에
닫혔다.** 키트는 이제 **#23(공유 맹점 가설 — 판정 충돌의 판별)**을 시험하는 도구다:
닫힘 술어가 "추가 교차 모델 red-team ≥2회(상이 개발사 포함, 동일 키트)"이므로,
GPT·Llama·DeepSeek 등 **Gemini 이외의 개발사 모델**로 반복 실행하는 것이 다음 단계다.
(c)가 재현되면 공유 맹점 확증, 재현되지 않으면 분산/기소 편향으로 닫힌다.
**2-pass 규칙(회차 62)**: 검증관은 제출 전 자체 팩트체크 재심을 의무 수행하고 최종본만
제출한다 — 사후(저자 요청) 재심은 표본 오염(보고서 #4 선례). 재현 시험은 2-pass 완료본만 계상.

**요건(외주 7회차 정밀화 — #23에도 동일 적용)**: 제출물은 **적대적 임무 하에**(`PROMPT.md`
또는 동등물) **결함 목록과 등급 판정을 산출**해야 한다.
**주의: 적대적 프롬프트 없는 Deep Research형 종합 분석은 요건을 충족하지 않는다**
(외주 7회차 판정 — 외부 검토 #1(`redteam-archive/GEMINI-REVIEW-1.md`)이 바로 그 사례).

## 실행 절차

1. **다른 기반 모델**의 새 세션을 연다 — 예: OpenAI GPT 계열, Google Gemini 계열,
   Meta Llama 계열, DeepSeek, 또는 인간 검토자. **Claude 계열은 이 감사에 무효**다
   (프롬프트가 모델에게 자기 계열을 밝히고 해당 시 중단하도록 지시한다).
2. `PROMPT.md`의 코드 블록 안 프롬프트를 복사해 붙여넣는다.
3. `SYNTHESIS-r62.md`와 `worldview-r62.json`을 **파일 첨부**한다.
   첨부가 안 되는 모델이면 두 파일의 내용을 본문에 이어 붙여넣는다
   (길이 제한에 걸리면 `SYNTHESIS-r62.md` 전문 + `worldview-r62.json`의
   `meta`·`axioms`·`core_credences`·`extension_credences`·`open_problems` 관련 부분 우선).
4. 결과 보고서가 나오면 **전문을 그대로** 이 저장소의
   `redteam-archive/RED-TEAM-REPORT-4.md`(다음 빈 번호)로 저장한다 — **원문 보존 의무**(회차 57 신설,
   요약본 대체 금지).
5. **판정 외주**(RED-TEAM.md 프로토콜): 저자 세션은 결함 목록을 판정 없이 정리만 하고,
   별도의 독립 판정 세션이 각 결함의 수용/기각과 하향 폭(5포인트 격자)을 결정한다.
   저자 세션은 집행만 한다.
6. 집행 내역을 새 회차로 `data/worldview.json`·`SYNTHESIS.md`·`index.html`·`README.md`에
   기록하고, #23의 소급 채점 데이터로 등록한다(판정 (c) 재현 여부가 핵심 관측값).

## 주의

- 스냅샷은 **회차 62 시점**이다. 저장소의 최신 회차가 더 나아갔다면 원본
  (`SYNTHESIS.md`, `data/worldview.json`)에서 스냅샷을 다시 떠서 교체한 뒤 실행하라 —
  구판을 감사한 전례가 이미 2건(보고서 #2는 회차 44 시점, 보고서 #3은 회차 43 시점 구판 대상)이다.
  감사 시작 전 대상의 `meta.last_cycle` 확인은 프롬프트에도 지시돼 있다.
- 이 감사가 결함을 하나도 못 찾아도 #18은 닫힌다(닫힘 술어는 "실행"이지 "결함 발견"이
  아니다). 단, "못 찾았다"는 보고서도 원문 보존 대상이다.
